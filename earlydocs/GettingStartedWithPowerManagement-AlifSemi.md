# How to get started with Power Management

## Introduction

Edge devices—from consumer wearables and smart home sensors to industrial IoT endpoints and wireless sensor networks—are increasingly battery-powered yet compute-intensive. These devices must run AI/ML workloads locally, respond to real-time sensor events, and maintain wireless connectivity, all while operating for months or years on a single battery. Without effective power management, even the most capable hardware becomes impractical for deployment in the field.

### Why Power Management Matters

Alif Semiconductor's Ensemble and Balletto families of processors are designed from the ground up for energy-efficient operation in battery-powered environments. Unlike traditional always-on processors that must stay plugged into a power source, Alif devices use energy-efficient cores capable of waking up to perform tasks and sleeping to save energy. However, unlocking this potential requires deliberate power management at both the hardware and firmware level.

Key challenges that power management addresses include:

- **Battery Life**: Efficient power management directly extends the operational lifespan of battery-powered IoT devices.

- **Performance / Efficiency Balance**: Alif's heterogeneous architecture lets you run demanding ML workloads on Cortex-M55/Ethos-U55 while keeping always-on tasks on energy-sipping Cortex-M55 HE cores.

- **System Cost**: Lower power budgets translate to smaller batteries, simpler thermal designs, and reduced BOM cost.

- **Always-On Intelligence**: With the right power strategy, devices can continuously monitor sensors, run keyword spotting, or perform anomaly detection—without draining the battery.

### Alif's Power Management Architecture at a Glance

Alif Ensemble and Balletto families feature the **Autonomous intelligent Power Management (aiPM)** subsystem, which autonomously transitions the SoC through several power modes:

| **Mode**    | **Description**                                              |
|-------------|--------------------------------------------------------------|
| **GO**      | Full run — all requested power domains and clocks active     |
| **READY**   | CPUs in Wait-for-Interrupt (WFI); peripherals still clocked  |
| **IDLE**    | CPUs clock-gated; most peripherals clock-gated               |
| **STANDBY** | All non-AON domains powered down; SRAM retained selectively  |
| **STOP**    | Deepest sleep — only VBAT domain alive; wake via RTC or GPIO |

This document guides the user through various resources available from understanding, evaluating, fine tuning, to finally implementing for user applications.

## Fundamentals

Before diving into the hands-on resources in this guide, we recommend reading the [**<u>Alif Processors Power Saving Features</u>**](https://alifsemi.com/download/AWPR0002) (Document: AWPR0002 v2.0) white paper to build a solid foundation in power management for Alif devices. The white paper covers:

- Operating regions (High-Performance, High-Efficiency, and Always-On)

- CPU states (RUN, SLEEP, OFF) and SoC power modes (GO, READY, IDLE, STANDBY, STOP)

- Alif's Autonomous Intelligent Power Management (aiPM) architecture

- Power Management Unit (PMU), retention memory options, and selectable clock frequencies

- Wakeup sources, latency considerations, and design best practices

## Power Estimation

The [Alif Conductor tool](https://conductor.alifsemi.com) is a web-based configuration and power estimation tool for Alif Ensemble and Balletto devices. It allows users to estimate power consumption by configuring active power domains, clocks, peripherals, and power modes for a given use case.

**What it does:**

- Allows user to configure various options for different power use cases in the life cycle of the product and estimate power consumption in each use case and battery life based on the duty cycle of each case.

- Calculates expected current consumption (mA) and power draw (mW) based on selected device configuration — active cores, clock frequencies, peripherals, and power modes.

- Supports both Ensemble and Balletto devices.

- Generates a Device Configuration JSON file (for static boot-time configuration via SETOOLS) and C header files (clocks.h and resources.h) for use in application code.

**Known Limitations:**

- PD6 SYSTOP power draw is currently overstated by approximately 2× in the web tool.

- mA and mW values are only accurate for the 1.8V input voltage setting. The tool displays constant current across supply voltages, whereas in reality current should decrease at higher voltages because the core voltage is generated by a buck DCDC (not an LDO).

- A spreadsheet version of the estimator is available that handles voltage scaling correctly and is recommended for more accurate estimates until the web tool issues are resolved. Please reach out to Alif Support if you would like the spreadsheet version.

**Note:** The Conductor power estimator provides coarse first-pass estimates. More accurate power estimation requires factoring in the specific use case, duty cycle, and optimization applied to the application firmware.

[Conductor Tool](https://conductor.alifsemi.com)

## Bare-Metal Low Power Example Projects on GitHub

The following bare-metal project repositories are available on [GitHub under the alifsemi organization](https://github.com/alifsemi). These projects provide hands-on, ready-to-build examples for implementing low power modes and peripherals on Alif Ensemble and Balletto processors. All projects are written in C and do not use an RTOS — they run directly on the hardware (bare metal). These projects are a good starting point to use as a template based on the customer product use case.

### alif_LowPowerDemo_StopMode

A multicore template project showcasing entry into and exit from STOP power mode. In STOP, only the VBAT / always-on domain remains powered; the system can wake via RTC alarm or a configured GPIO, or another peripheral in the always-on domain. The demo focuses on minimizing wake latency, and on orderly restoration of clocks, PLLs, and required peripherals after wake before resuming application work.

🔗 [alifsemi/alif_LowPowerDemo_StopMode](https://github.com/alifsemi/alif_LowPowerDemo_StopMode)

### alif_LowPowerDemo_StandbyMode

A multicore template project demonstrating STANDBY power mode transitions, core-to-core communication via the Message Handling Unit (MHU), and wakeup events. At power-on, the HE core configures the LPTIMER as a wakeup source and puts the MCU into global STANDBY Mode. On each LPTIMER wakeup (every 1000 ms), only the HE core wakes to perform a task. Every tenth wakeup, the HE core uses MHU to bring up the HP core for additional processing. Supports Ensemble E1, E1C, E3–E8, and Balletto B1.

🔗 [alifsemi/alif_LowPowerDemo_StandbyMode](https://github.com/alifsemi/alif_LowPowerDemo_StandbyMode)

### alif_LowPowerDemo_AudioCapture

A single-core template project demonstrating the minimum device configuration for continuous audio capture at low power. The demo captures audio data from a pair of external I2S microphones while reducing power consumption through clock and power configuration adjustments via the SERVICES aiPM API — clock frequencies are reduced by running the MCU from RC rather than PLL and unused power domains are disabled.

🔗 [alifsemi/alif_LowPowerDemo_AudioCapture](https://github.com/alifsemi/alif_LowPowerDemo_AudioCapture)

### alif_LowPowerDemo_ImageCapture

A template project for GPIO-triggered image capture with dynamic frequency scaling to reduce system-level power when idling between frame captures. The demo manages transitions between active and idle states: during capture, clocks are sourced from PLL at high frequency; during idle, clocks switch to 4.8 MHz HFRC/HFXO and the PLL is disabled. Camera peripherals (CPI, CSI2) are clock-gated and the D-PHY is power-gated when idle. Supported on Ensemble E7 and E8 DevKits.

🔗 [alifsemi/alif_LowPowerDemo_ImageCapture](https://github.com/alifsemi/alif_LowPowerDemo_ImageCapture)

### alif_LowPower_SensorSampling

A use-case based low power demo for Alif Ensemble processors demonstrating how to continuously capture sensor data from an external SPI ADC at low power. The project uses DMA and hardware event routing to drive SPI transfers without CPU interaction, maximizing CPU sleep time. It also implements a dynamic clock scaling scheme that switches between low-frequency HFXO and high-frequency PLL sources to optimize power draw based on application need — running data processing at high frequency and data capture at low frequency. Supported on Ensemble devices only.

🔗[alifsemi/alif_LowPower_SensorSampling](https://github.com/alifsemi/alif_LowPower_SensorSampling)

## Fine Tuning and Debugging

### SERVICES_aiPM_CLI

A command-line interface for interacting with Alif SERVICES aiPM (power management). It lets you inspect and configure power domains, clocks, and SoC power modes (GO/READY/IDLE/STANDBY/STOP), and trigger transitions for evaluation. Useful for bring-up, debugging, and validation when tuning low-power behavior on Ensemble and Balletto devices. The information gathered from this can then be used to configure the power profiles in the application.

[alifsemi/SERVICES_aiPM_CLI](https://github.com/AlifSemi/SERVICES_aiPM_CLI)
